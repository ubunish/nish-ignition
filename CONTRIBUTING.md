# Contributing

## Coding Style

- modular
- scalable
- consistent
- documented → illustrated diagrams + text

### Commits

Format: `prefix: short phrase`. Lowercase prefix, no body.

Prefixes:

- `init` — initial scaffolding of a new doc or folder
- `feat` — new content or new page
- `fix` — correction to existing content
- `docs` — edits to existing markdown that aren't fixes (rewrites, restructures, additions)
- `refactor` — reorganising without changing meaning
- `chore` — housekeeping (gitignore, file moves, dependency bumps)

### Branches

- `main` — canonical, finished, merged
- `prefix: short phrase` - features, fixes, docs, refactors, chores

## Documentation

### Writing

Idea stated once. Understood reading once.

- *audience* - graduate engineer
- *exclude* - articles, conjunctions, examples
- *format* - diagrams > text
- *headings & short text* - title case
- *length* - concise
- *technical* - simple > complex
